# Security Policy

## Supported Versions

| Version | Supported |
| --- | --- |
| 1.x | Yes |
| < 1.0.0 | No |

## Reporting a Vulnerability

If you discover a security issue in AFKBOT, do not open a public issue first.
Send a private report with reproduction details, impact, and affected versions
to `security@afkbot.io`.

## What to Include

- A clear description of the issue
- A severity assessment
- A description of the affected component
- Steps to reproduce
- Expected and actual behavior
- The demonstrated impact
- Affected configuration, environment, or version
- Any proof-of-concept details needed to validate the report
- Suggested mitigation or remediation if you have one

## Response Process

- We will acknowledge receipt as soon as practical.
- We may ask follow-up questions to reproduce or scope the issue.
- Public disclosure should wait until a fix or mitigation is available.
- Reports without actionable reproduction detail may be deprioritized.

## Scope

Please report vulnerabilities related to:

- authentication and authorization
- credential storage or leakage
- remote code execution paths
- sandbox or tool boundary bypasses
- sensitive data exposure
- installer, updater, and local runtime privilege boundaries
